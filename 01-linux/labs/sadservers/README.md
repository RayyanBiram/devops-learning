Linux troubleshooting scenarios on real servers - diagnose and fix broken systems in a capture-the-flag format.

## Skills practiced
 
- Diagnosing and fixing broken Linux servers under time pressure
- Process management, disk usage, and log analysis
- Service troubleshooting (`systemd`, `nginx`, `ssh`, firewall)
- Networking and DNS debugging
- Reading logs (`journalctl`, `/var/log/`, `tail -f`)
